# CLAUDE - 東京路線図アプリ プロジェクト概要

## プロジェクト概要
このプロジェクトは東京の電車路線図を表示し、駅間の経路検索機能を提供するWebアプリケーションです。

### 技術スタック
- **フロントエンド**: Astro + React + TypeScript
- **地図表示**: Leaflet + React-Leaflet
- **開発サーバー**: `npm run dev` (localhost:8080)
- **ビルド**: `npm run build`

### プロジェクト構成
```
src/
├── components/           # Reactコンポーネント
│   ├── RailwayMap.tsx   # メイン地図コンポーネント
│   ├── StationSelector.tsx    # 駅選択UI
│   ├── RouteRecommendations.tsx # 経路推薦表示
│   ├── SimpleMap.tsx    # シンプル地図表示
│   └── DataTest.tsx     # データテスト用
├── data/                # 路線データ（39路線）
│   ├── routes.ts        # 全路線統合定義
│   ├── yamanote.ts      # 山手線データ
│   ├── chuo.ts          # 中央線データ
│   ├── ginza-line.ts    # 銀座線データ
│   └── ...              # その他路線データ
├── utils/
│   └── routeFinder.ts   # 経路検索アルゴリズム
└── pages/
    └── index.astro      # メインページ
```

### 主要機能
1. **インタラクティブ路線図表示**
   - 39路線の東京電車路線（JR、地下鉄、私鉄、新交通システム）
   - ズームレベル対応駅表示
   - 路線の表示/非表示切り替え
   - 駅クリック時の通過路線情報表示
   - お台場エリア（りんかい線・ゆりかもめ）および首都圏広域（つくばエクスプレス）をカバー
   - 神奈川県方面（南武線・根岸線）の重要路線を追加

2. **経路検索機能**
   - 出発駅・到着駅選択
   - 最適経路推薦（最短時間、乗換回数考慮）
   - 複数経路候補表示

3. **UI機能**
   - 乗換駅のみ表示フィルター
   - 折りたたみ可能な操作パネル
   - レスポンシブデザイン

4. **所要時間フィルター機能（現在実装中）**
   - 出発駅から指定時間内にアクセス可能な駅のみを表示
   - 5分〜60分の範囲で設定可能
   - 乗換を含む最短経路時間で計算
   - デバッグ用の一致性チェック機能付き

### データ構造
- 各駅は緯度経度座標を持つ
- 路線は駅の配列として定義
- 駅間の移動時間は距離ベースで算出

### 路線カバレッジ分析（39路線）

#### JR線（13路線）
- **環状・幹線**: 山手線（31駅）、中央線、京浜東北線、東海道線、総武線
- **郊外・広域**: 常磐線、埼京線、高崎線、東海道本線、武蔵野線、横浜線
- **神奈川方面**: 南武線（立川〜川崎）、根岸線（大船〜横浜）

#### 東京メトロ（9路線）
- 銀座線、丸ノ内線、日比谷線、東西線、千代田線
- 有楽町線、半蔵門線、南北線、副都心線

#### 都営地下鉄（4路線）
- 浅草線、三田線、新宿線、大江戸線

#### 私鉄（10路線）
- 小田急線、小田急江ノ島線、京王線
- 東急東横線、東急田園都市線
- 西武池袋線、西武新宿線、東武東上線
- 京急線、京成本線

#### 市営地下鉄・新交通システム（3路線）
- **横浜市営地下鉄ブルーライン**: 横浜エリア
- **東京モノレール**: 浜松町〜羽田空港
- **りんかい線**: 大崎〜新木場（お台場エリア）

#### 新交通システム・広域鉄道（2路線）
- **ゆりかもめ**: 新橋〜豊洲（お台場観光・ビジネスエリア）
- **つくばエクスプレス**: 秋葉原〜つくば（首都圏広域高速鉄道）

**カバレッジ特徴:**
- **都心部**: 山手線・地下鉄網による完全カバー
- **観光地**: お台場エリア（りんかい線・ゆりかもめ）を完全接続
- **広域エリア**: 茨城県つくば（つくばエクスプレス）まで拡大
- **神奈川方面**: 南武線・根岸線で川崎・横浜方面を強化
- **乗換最適化**: 主要ターミナル駅での効率的な接続
- **総駅数**: 約400駅以上をカバー

## コミットに関する重要な規則

### 1. 確認なしでのコミットを禁止
- **勝手にコミットをしないこと**
- コミットする前には必ずユーザーに確認を求めること
- ユーザーが明示的にコミットを依頼した場合のみ実行すること

### 2. 論理的な単位でのコミット
- **まとめてコミットしない**
- 一つの論理的な変更単位ごとにコミットすること
- 関連のない変更は別々のコミットに分けること
- 各コミットは単独で理解可能で、意味のある変更を含むこと

### 3. コミットメッセージ
- 変更内容を簡潔に説明すること
- 何を変更したかではなく、なぜ変更したかを重視すること

## 例
良い例：
- 機能追加のコミット
- バグ修正のコミット  
- リファクタリングのコミット

悪い例：
- 複数の無関係な変更をまとめたコミット
- ユーザーの許可なく実行されたコミット
